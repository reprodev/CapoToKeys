# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` release and active development on `dev`.

## Reporting a Vulnerability

Please report vulnerabilities privately and do not open public issues for security disclosures.

Preferred contact:

- Open a private GitHub security advisory in this repository, or
- Email the maintainer listed in the repository profile.

Include:

- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

You will receive acknowledgment as soon as possible, followed by triage and fix status updates.
